0.6

Appearance

  • redesign the song library
  • move song and songbook panels into preferences
  • new checkbox mechanism for song selection
  • new tab in preferences dialog for song editor options

Performances

  • remove qt's deprecated filesystem watcher for performance and stability
  • remove the sqlite database that was storing the library
  • delay the initialisation of the library after the startup of the application

Misc

  • new cmake toolchain
  • debian packaging with git-buildpackage
  • restore state for file dialogs
  • rename dialogs according to their triggering action
  • monospace font for song editors
  • new icon set for MacOs
  • allow custom commands for make/clean process
  • new dialog for clean/cleanall operations

0.5

Cross-platform

  • Windows: initial support. Note that a LaTeX distribution and a Python installation is required to produce pdf
  • Mac: bug fixes and better integration
  • Debian: move songbook dependencies from required to recommended
  • Gentoo: update ebuild dependencies

Appearance

  • new icons
  • tango icon theme for Mac and Windows
  • better integration for toolbars
  • improved form based dialogs
  • better placement of elements in menus
  • fewer message box to simplify user experience
  • improved columns view
  • new icon+text view for the library's album column

Features

  • rewrite the download process of a library. It now uses libarchive instead of an external system call to git
  • it is now possible to configure a proxy for downloading a library
  • auto-completion in the filter bar
  • several keyshortcuts have been corrected
  • new indentation mode in the song editor which now behaves like emacs: when "Tab" is pressed, the current line is correctly indented instead of inserting a tabulation
  • remove "Tools" menu since these actions were not relevant in the gui
  • auto resize cover when starting a new song

Performances

  • library rebuild operation drastically improved (30x faster!)
  • startup time reduced
  • improved sql queries

Bugs

  • many bugs have been fixed
  • use the github issues tracker tool

In progress

  • start working on separating the logic and appearance in the application
  • start evolving the library model
